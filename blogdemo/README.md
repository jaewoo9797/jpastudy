# 관계형 데이터 모델링 학습 계획 (블로그 시스템)

블로그 시스템 프로젝트를 진행하면서 관계형 데이터 모델링을 체계적으로 학습

## 목표 설정
학습 목표
- JPA를 사용한 관계형 데이터 매핑 이해 및 구현
- 1:N , N:1, N:M 등 다양한 관계 모델링 연습.
- 양방향 연관관계와 순환 참조 문제 해결 방법 학습.
- 연관 데이터를 효율적으로 조회하고 처리하는 방법 이해.

## 학습 단계
1. 사전준비
    - 관계형 데이터베이스 설계 기초
      - 엔티티와 테이블의 관계
      - 1:N, N:1, N:M 관계
    - JPA 주요 어노테이션
      - Entity
      - Id
      - GeneratedValue
      - OneToMany
      - ManyToOne
      - JoinColumn
      - MappedBy 와 양방향 연관관계
      - 지연로딩과 즉시로딩


## 기능 요구사항 설계

간단한 글을 올리는 게시판 프로젝트 진행

### USER
- Create
  - 새로운 유저의 회원가입
- Update
  - 유저의 정보를 수정
    - 비밀번호 변경 -> 비밀번호를 재입력 시키고 싶음
- SELECT
  - 유저의 정보를 조회
    - 로그인
    - 마이페이지 -> 자신의 쓴 글 목록, 자신이 쓴 댓글 목록 어떻게 ?
    - 비밀번호가 맞는지 체크하기 - 자신의 정보를 수정하기 위한 페이지로 넘어가기전, 확인했는지 검증을 어떻게할것인가?
- DELETE
  - 유저의 정보를 삭제
    - 소프트 삭제

### ARTICLE
- CREATE
  - 새로운 글 작성
- UPDATE
  - 글의 정보를 수정
- SELECT
  - 특정 글을 조회
  - 모든 글을 조회
  - 모든 글을 페이징 처리
- DELETE
  - 글의 정보를 삭제

### COMMENT
- CREATE
  - 새로운 댓글 작성
- UPDATE
  - 댓글의 정보를 수정
- SELECT
  - 모든 댓글을 조회
- DELETE
  - 댓글 정보를 삭제
