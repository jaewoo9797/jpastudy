시큐리티를 적용해서 정적파일이 보안에 걸려서 로드하지 못하는 에러가 발생한다. 이는 스프링부트 서버 사이드 웹 애플리케이션을 만들고 있어서 이런 에러가 발생한다.

이런 문제를 해결하기 위해서 간단하게 해결하는 방법을 찾아서 적용하다가, 갑자기 모든 정적파일이 시큐리티에 막혀 당황했다.
앞으로 스프링부트로 화면ㄱ ㅜ성을 할지는 잘 모르겠지만, 스프링 시큐리티는 쓸 예정이니 간단하게 정리를 해두겠다.

문제 발생 코드

```java
시큐리티 설정 파일 내
    public void configure(WebSecurity web) throws Exception {
        web.ignoring().requestMatchers(PathRequest.toStaticResources().atCommonLocations())
                .requestMatchers(PathRequest.toH2Console())
        ;
    }
```

이렇게 간단하게 적용하면 해결될거라고 생각을 했다. 나름 여기저기 찾아다녀 보면서 서치한 결과로 이게 가장 간단한 방법이라고 생각했다.

보다 서버에 성능을 좋게하기 위해서 시큐리티 필터 로직에 들어가기 전에 허용해주도록 하는 것인줄 알았는데, 적용이 하나도 안되었다.

이를 해결하는 방법으로는 WebSecurityCustomizer 라는 인터페이스를 빈으로 등록해야한다.

```java
    @Bean
    public WebSecurityCustomizer webSecurityCustomizer() {
        return web -> web.ignoring()
                .requestMatchers(PathRequest.toStaticResources().atCommonLocations())
                .requestMatchers(PathRequest.toH2Console())
                .requestMatchers("/fonts/**", "/images/**");
    }
```

이렇게 빈으로 등록하는 코드를 작성해두면 기존의 정적 파일에서 생기던 문제를 해결할 수 있다.
